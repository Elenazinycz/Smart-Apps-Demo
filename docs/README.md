# Dokumentation

Dieses Projekt folgt einer SOLO-Variante der Methodik aus `jacekzawisza/modus-operandi`.

## Struktur

| Datei | Zweck |
|---|---|
| `spec.md` | Zentrale Produktgrundlage; ersetzt `prd.md` |
| `architecture.md` | Technische Leitplanken und Architekturstand |
| `backlog.md` | Feature-Registry mit stabilen IDs |
| `decisions.md` | Chronologisches Log wichtiger Produkt- und Architekturentscheidungen |
| `modus-operandi.md` | Projektspezifische Arbeitsweise |

## Bewusst nicht verwendet

- `prd.md`: Rolle wird von `spec.md` uebernommen.
- `docs/meetings/`: Solo-Projekt, keine Meeting-Ablage noetig.
- `docs/results/`: Solo-Projekt, aktuell keine separate Ergebnis-Schicht noetig.
